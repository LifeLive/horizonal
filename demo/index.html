What is it?

Horizonal.js takes any static HTML document and transform it into a series of discrete "pages". The transitions between one page an the next can then be programmed via CSS transforms, CSS animations, JavaScript, or any combination of these.

Disable Horizonal with the switch above and you'll see that this is just a regular old HTML document. Try out some of the themes to get a taste of the kind of effects that can be achieved.

How?

You specify which elements you wish to display via the familiar jQuery selector syntax. Horizonal will then traverse the document, splitting it up into a series of pages, and figuring out which element should go where.

When you move from one page to the next, Horizonal alters the elements' class, allowing you to specify CSS-based transition effects. It also provides hooks for triggering Javascript callbacks, allowing for even more sophisticated effects.

Features

  • Fully customizable - limited only by your imagination (and JavaScript & CSS chops)
  • Touch-enabled: page through the document by swiping.
  • Can handle URL fragments - e.g. go to that poem
  • Can handle large elements that span multiple pages - e.g. this huge photo

Quick Start

  1. Requires jQuery <= 1.8, so make sure you have included that on your page.
  2. Include the horizonal.min.js after jQuery.
  3. Include the horizonal.css file in your document head.
  4. Run horizonal.init();

Documentation

The above describes only the most basic usage. To get some cool transitions going, you'll want to experiment with both the CSS style definitions, and the Horizonal config. Both are described in more detail in the GitHub repo, and there you can also take a look at the CSS and configuration behind each of the example themes.
